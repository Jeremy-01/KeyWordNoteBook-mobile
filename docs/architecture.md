## 系统概览

### 项目背景

KeyWordNoteBook 是一款安全的密码本管理器，目标是：
- 为用户提供跨平台（iOS/Android）的密码管理体验
- 确保数据的端到端加密安全性
- 支持多设备实时同步

### 核心特性

| 特性 | 说明 |
|------|------|
| 端到端加密 | AES-256-GCM + Argon2 密钥派生 |
| 生物识别 | 指纹/面容快速解锁 |
| 云端同步 | 多设备数据一致性 |
| 离线支持 | 本地优先，智能同步 |

### 技术栈

#### 移动端

| 组件 | 技术选型 | 替代方案 | 选择理由 |
|------|---------|---------|---------|
| 框架 | Flutter 3.x | React Native | 跨平台一致性好，性能优秀 |
| 状态管理 | Riverpod | Provider, BLoC | 类型安全，编译时检查 |
| HTTP | Dio | http 包 | 拦截器，适配器，复用连接 |
| 本地存储 | SQLite | Hive, Drift | 结构化数据，事务支持 |
| 安全存储 | flutter_secure_storage | Keychain | 跨平台统一 API |
| 生物识别 | local_auth | flutter TouchID | 官方维护 |

#### 后端

| 组件 | 技术选型 | 替代方案 | 选择理由 |
|------|---------|---------|---------|
| 框架 | FastAPI | Flask, Django | 自动 OpenAPI，高性能 |
| ORM | SQLAlchemy | Prisma, Tortoise | 成熟稳定 |
| 数据库 | SQLite | PostgreSQL, MySQL | 零配置，适合中小规模 |
| 认证 | JWT | Session | 无状态，扩展性好 |
| 密码哈希 | Argon2 | bcrypt, scrypt | 最佳安全参数 |

### 模块职责

#### Core 层

| 模块 | 职责 | 对外接口 |
|------|------|---------|
| `crypto/` | 数据加密、密钥管理、密码强度 | `CryptoService` |
| `network/` | API 调用、Token 管理、错误处理 | `ApiClient`, `TokenManager` |
| `storage/` | 数据库操作、数据持久化 | `LocalStorage` |
| `sync/` | 数据同步、冲突处理 | `SyncService` |
| `biometric/` | 指纹/面容认证 | `BiometricService` |

#### Data 层

| 模块 | 职责 |
|------|------|
| `models/` | 定义数据结构，与 JSON/Database 映射 |
| `providers/` | 状态管理，连接 UI 和数据源 |
| `repositories/` | 抽象数据操作，隔离 Core 层 |

#### Features 层

| 模块 | 职责 |
|------|------|
| `auth/` | 登录、注册、Token 刷新 |
| `home/` | 密码列表、搜索、导航 |
| `item/` | 密码详情、编辑、删除 |
| `settings/` | 用户设置、同步配置 |

## 安全设计

### 密钥派生流程

```
┌─────────────────────────────────────────────────────────────┐
│                      用户主密码                               │
│                    (Master Password)                         │
└────────────────────────────┬────────────────────────────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
          ▼                  ▼                  ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  Argon2id       │ │  Argon2id       │ │  Argon2id       │
│  (本地存储密钥)  │ │  (加密密钥)     │ │  (HMAC 密钥)    │
└────────┬────────┘ └────────┬────────┘ └────────┬────────┘
         │                  │                  │
         ▼                  ▼                  ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│ AES-256 Key     │ │ AES-256 Key     │ │ HMAC-SHA256 Key │
│ (本地数据加密)  │ │ (云端数据加密)   │ │ (完整性校验)    │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### Argon2 参数配置

| 参数 | 移动端 | 后端 | 说明 |
|------|--------|------|------|
| memory_cost | 131072 KB | 131072 KB | 内存消耗 |
| time_cost | 6 | 6 | 迭代次数 |
| parallelism | 6 | 6 | 并行线程数 |
| hash_len | 64 | 64 | 输出长度 |

### 双重验证机制

对于敏感操作（查看密码、修改密码、删除密码）：

```
┌─────────────────────────────────────────────────────────────┐
│                        请求                                │
│  Authorization: Bearer <access_token>                      │
│  X-Master-Password: <master_password>                      │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                    第一层验证                               │
│                    JWT Token 验证                           │
│                    ↓ 失败 → 401 Unauthorized                │
│                    ↓ 成功                                  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                    第二层验证                               │
│                    主密码验证                                │
│                    ↓ 失败 → 403 Forbidden                   │
│                    ↓ 成功                                  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                    执行敏感操作                              │
└─────────────────────────────────────────────────────────────┘
```

### 数据存储安全

| 存储位置 | 存储内容 | 加密方式 |
|---------|---------|---------|
| SecureStorage | JWT Token, 加密密钥 | 操作系统 Keychain/Keystore |
| SQLite | 密码条目、设置 | AES-256-GCM |
| 云端数据库 | 加密后的密码数据 | AES-256-GCM |

## 同步机制

### 同步策略: 增量同步 + 版本向量

```
┌─────────────────────────────────────────────────────────────┐
│                       同步状态                               │
│                                                             │
│  ┌─────────────┐              ┌─────────────┐             │
│  │   移动端    │              │    后端     │             │
│  │  version:5  │              │  version:7  │             │
│  └──────┬──────┘              └──────┬──────┘             │
│         │                            │                     │
│         │    GET /sync/status         │                     │
│         │═════════════════════════════▶                     │
│         │                            │                     │
│         │◀════════════════════════════│                     │
│         │    后端版本:7 > 移动端版本:5│                     │
│         │    需要拉取                   │                     │
│         │                            │                     │
│         │    POST /sync/pull          │                     │
│         │═════════════════════════════▶                     │
│         │                            │                     │
│         │    获取 version 6,7 的变更   │                     │
│         │◀════════════════════════════│                     │
│         │                            │                     │
│         │    更新本地数据              │                     │
│         │    version: 5 → 7           │                     │
│         │                            │                     │
└─────────────────────────────────────────────────────────────┘
```

### 冲突解决策略: Last-Write-Wins

```
┌─────────────────────────────────────────────────────────────┐
│                      冲突场景                               │
│                                                             │
│  设备 A: 修改密码 "website"  @ 10:00:30                      │
│  设备 B: 修改密码 "website"  @ 10:00:25                      │
│                                                             │
│  由于 10:00:30 > 10:00:25，设备 A 的修改优先                │
│                                                             │
│  服务端保存: 10:00:30 的版本                                 │
└─────────────────────────────────────────────────────────────┘
```

### 离线优先策略

```
┌─────────────────────────────────────────────────────────────┐
│                    操作执行顺序                              │
│                                                             │
│  1. 立即写入本地数据库                                       │
│  2. 标记为 "pending_sync"                                   │
│  3. 后台尝试同步                                            │
│  4. 同步成功后清除标记                                       │
│  5. 同步失败时保留，等待下次重试                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 技术决策记录

### ADR-001: 使用 Flutter 作为移动端框架

**日期**: 2026-05-10
**状态**: 已通过
**决策者**: 技术团队

**背景**: 需要开发跨平台移动应用

**考虑选项**:
- Flutter
- React Native
- Kotlin Multiplatform (KMP)
- 原生开发 (iOS Swift + Android Kotlin)

**决策**: 选择 Flutter

**理由**:
1. 单代码库覆盖 iOS/Android
2. Dart 语言学习曲线平缓
3. 丰富的生态和社区支持
4. 优秀的热重载体验
5. 性能接近原生应用

**后果**:
- 需要学习 Dart 语言
- 某些平台特定功能需要平台通道

### ADR-002: 使用 FastAPI 作为后端框架

**日期**: 2026-05-10
**状态**: 已通过
**决策者**: 技术团队

**背景**: 需要轻量级、高性能的 REST API 服务

**考虑选项**:
- FastAPI
- Flask
- Django REST Framework
- Express.js

**决策**: 选择 FastAPI

**理由**:
1. 自动生成 OpenAPI/Swagger 文档
2. 异步支持，高并发性能
3. Pydantic 数据验证
4. 类型安全（可与 TypeScript/Flutter 配合）
5. 现代化 Python 异步生态

---

### ADR-003: 使用 Argon2 作为密钥派生函数

**日期**: 2026-05-10
**状态**: 已通过
**决策者**: 技术团队

**背景**: 需要安全的密钥派生方案

**考虑选项**:
- Argon2id
- PBKDF2
- scrypt
- bcrypt

**决策**: 选择 Argon2id

**理由**:
1. 2015 年密码哈希竞赛冠军
2. 内存硬特性，对 GPU 攻击有抵抗力
3. 现代安全标准推荐
4. 参数可调节，适应未来硬件

---

### ADR-004: 使用 Riverpod 进行状态管理

**日期**: 2026-05-10
**状态**: 已通过
**决策者**: 技术团队

**背景**: 需要可靠的状态管理方案

**考虑选项**:
- Riverpod
- Provider
- BLoC
- GetX

**决策**: 选择 Riverpod

**理由**:
1. 编译时安全检查
2. 依赖注入内置
3. 测试友好
4. 易于理解和维护
5. 官方推荐的新一代方案

---

### ADR-005: 使用 JWT 无状态认证

**日期**: 2026-05-10
**状态**: 已通过
**决策者**: 技术团队

**背景**: 需要可扩展的认证方案

**考虑选项**:
- JWT (无状态)
- Session (有状态)

**决策**: 选择 JWT

**理由**:
1. 无需服务端存储会话
2. 易于水平扩展
3. 支持多设备登录
4. Token 可包含用户信息

**权衡**:
- Token 过期需要刷新机制
- Token 泄露风险（需要 HTTPS）
