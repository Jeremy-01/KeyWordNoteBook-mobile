## 一、项目概述

### 1.1 背景与目标

KeyWordNoteBook（密码本管理器）从 PyQt5 桌面端迁移至移动端（iOS/Android），引入账号系统和云端同步功能，实现多设备数据互通。

### 1.2 核心需求

| 需求类型 | 描述 | 状态 |
|---------|------|------|
| 移动端支持 | Flutter iOS/Android 应用 | ✅ 已完成 |
| 账号系统 | 用户注册、登录、密码找回 | ✅ 已完成 |
| 端到端加密 | AES-256-GCM + Argon2 | ✅ 已完成 |
| 云端同步 | 密码本数据云端加密存储 | ✅ 已完成 |
| 生物识别 | 指纹/面容快速解锁 | ✅ 已完成 |

### 1.3 技术选型

| 组件 | 技术方案 | 说明 |
|-----|---------|------|
| 移动端框架 | Flutter 3.x | 跨平台 iOS/Android |
| 状态管理 | Riverpod | Flutter 状态管理 |
| 网络 | Dio | HTTP 客户端 |
| 本地存储 | SQLite + SecureStorage | 本地加密存储 |
| 加密 | Argon2 + AES-256 | 端到端加密 |
| 生物识别 | local_auth | 系统级指纹/面容认证 |
| 后端框架 | FastAPI | RESTful API |
| 数据库 | SQLite + SQLAlchemy | ORM 数据持久化 |

## 五、安全设计

### 5.1 密钥派生

```
用户主密码 (Master Key)
         │
         ├──► Argon2id ──► AES-256 Key (加密数据)
         │
         └──► Argon2id ──► HMAC-SHA256 Key (完整性校验)
```

### 5.2 Argon2 参数（客户端与服务端保持一致）

| 参数 | 值 |
|------|------|
| memory_cost | 131072 KB |
| time_cost | 6 |
| parallelism | 6 |
| hash_len | 64 |

### 5.3 数据安全

- 密码使用 AES-256-GCM 加密存储
- HMAC-SHA256 保证数据完整性
- JWT Token 时效性控制
- 敏感数据存储在 flutter_secure_storage
